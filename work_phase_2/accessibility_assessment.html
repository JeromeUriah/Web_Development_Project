Accessibility
Assessment

This page contains the accessibility assessment for this website

Applicable Criteria


1.1.1 Non-text Content

Users require videos and pictures that enhances learning.

Having alternative text descriptions for these items would enhance accessibility for users with visual impairments who use screen readers.


1.4.1 Use of Color

As there will be much content throughout the website. Colors would be used to differntiate between pages or sections.

Having colors will guide navigations and enhance understanding of information.


1.4.1 Resize Text

As many user will be using website on their mobile devices, having the text be scalable would impact the clarity of information

Allowing users to adjust text size to their needs and ensuring content remains legible regardless of screen size or user preference


3.2.4 Consistent Identification

Having consitent identification will allow all users, especially those with disabilities, to navigate and understand the website

Principle: Perceivable

Guidline 1.1 - Text Alternatives

1.1.1 Non-text Content


[Definition]

Having Alternative text descriptions for items that are not texts


[Action Taken]

Buttons and images will have alternative texts that describes the function or name

Guidline 1.3 - Adaptable

1.3.2 Meaningful Sequence


[Definition]

Content should be arrange in a way that would make sense to a user in terms of flow


[Action Taken]

All pages should have similar layouts with menu > content in a top-down format

Guidline 1.4 - Distinguishable

1.4.1 Use of Color


[Definition]

Components of website should be differentiated with the use of colors


[Action Taken]

All components have background colors that differenciates
them from the background

1.4.3 Contrast (Minimum)


[Definition]

Colors should have a contrast ratios of 4.5:1 for text and 3.1 for large text


[Action Taken]

Using adobe color wheel, colors are checked for contrasts agaisnts backgrounds

1.4.4 Resize Text


[Definition]

Text should scale with zoom


[Action Taken]

All text use REM as unit sizes to allows for scaling

Principle: Operable

Guidline 2.1 - Keyboard Accessible

2.1.1 Keyboard


[Definition]

Website should be able to be navigated with a keyboard


[Action Taken]

Buttons and components are tagged with ARIA tags to allows for "tab" navigation

Guidline 2.4 - Navigable

2.4.7 Focus Visible


[Definition]

Items selected with a keyboard input should have a visual indicator


[Action Taken]

Buttons and components are tagged with ARIA tags to allows for "tab" navigation

Principle: Understandable

Guideline - 3.1 Readable

3.1.5 Reading Level


[Definition]

Information should be presented in a format that is easy to understand in terms of wordings


[Action Taken]

All directions on how to use the website are displyed using simple english and not over-complicated

Guideline - 3.2 Predictable

3.2.3 Consistent Navigatione


[Definition]

Menus and navigation buttons should be in the
same positions throughtout the website


[Action Taken]

All layouts are consistent with navigations in the same area

3.2.4 Consistent Identification


[Definition]

Similar components should also be visually consistent and
this applies to items that have similar functions


[Action Taken]

All buttons and menu items are styled using variables to ensure consitent colors

Guideline - 3.3 Input Assistance

3.3.1 Error Identification


[Definition]

Any input with errors should be visually identified


[Action Taken]

Errors in forms will be highlighted in red

3.3.2 Labels


[Definition]

Inputs should have tooltips that guides users on how to input information


[Action Taken]

Each input in a form would have hover tooltips that explains what should be inputted

3.3.4 Error Prevention and Recovery


[Definition]

Submissions should be able to be double checked, reverted and be double confirmed


[Action Taken]

Forms will have a additional confirmations and forms cached and saved to allow editing
